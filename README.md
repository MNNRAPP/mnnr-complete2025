# MNNR

MNNR is a rail-neutral governance layer for agentic payments, providing AI-actor self-identification, mandate binding, policy enforcement, and audit exports designed to support EU AI Act Article 50 transparency and PSD3/PSR-aligned payment governance.

Current pilot materials are available directly from MNNR.

This repository may contain prior prototype materials that are not representative of the current enterprise governance product. MNNR is no longer positioned as consumer SaaS, API metering, generic subscription billing, or self-serve developer payments infrastructure.

## What MNNR Provides

AI-actor self-identification for agentic payment flows.

Mandate binding for scope, amount, and expiry checks.

Policy enforcement before transaction authorization.

Append-only policy logs for audit review.

CSV and JSON audit exports for internal governance and regulatory review.

## What MNNR Is Not

MNNR is not a payment rail.

MNNR is not a bank.

MNNR is not a replacement for PSP core infrastructure.

MNNR is not consumer SaaS.

MNNR is not generic API metering or subscription billing software.

MNNR does not provide legal advice or compliance certification.

## Compliance Caveat

MNNR provides technical controls designed to support transparency, governance, and auditability in agentic payment flows. MNNR does not guarantee regulatory compliance. Customers remain responsible for their own legal, regulatory, and compliance determinations.
