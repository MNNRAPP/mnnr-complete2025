# Additional Information
# This application follows OWASP security best practices.
# We take security seriously and appreciate responsible disclosure.
# 
# Vulnerability Disclosure Process:
# 1. Email security@yourdomain.com with details
# 2. We will acknowledge within 48 hours
# 3. We will provide a timeline for fix within 7 days
# 4. We will credit researchers (if desired) after fix is deployed
#
# Please DO NOT:
# - Open public GitHub issues for security vulnerabilities
# - Test on production systems without authorization
# - Access or modify user data
# - Perform denial of service testing
#
# In Scope:
# - Authentication bypass
# - Privilege escalation
# - SQL injection
# - XSS (cross-site scripting)
# - CSRF (cross-site request forgery)
# - Sensitive data exposure
# - Business logic vulnerabilities
#
# Out of Scope:
# - Social engineering attacks
# - Physical attacks
# - DoS/DDoS attacks
# - Spam or content injection
# - Issues in third-party services (Stripe, Supabase)
